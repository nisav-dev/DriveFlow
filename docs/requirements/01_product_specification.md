# DriveFlow — מסמך אפיון מוצר מלא

---

## 1. שם הפרויקט
**DriveFlow** — פלטפורמה אינטרנטית מודרנית להשכרת רכבים

---

## 2. תיאור המערכת

DriveFlow היא מערכת תוכנה מלאה לניהול השכרת רכבים, המיועדת לחברות השכרה בינוניות עד גדולות.
המערכת מספקת חווית משתמש מודרנית ומהירה, כולל:
- ממשק לקוח לחיפוש, השוואה והזמנת רכבים
- ממשק ניהול מלא לניהול הצי, הסניפים, ההזמנות והדוחות
- מנוע הזמנות מלא עם תשלום, אישור והיסטוריה
- ניהול תחזוקה, נזקים והחזרות רכב

המערכת נבנית כ-Web Application בארכיטקטורת MVC מבוססת Java, Spring Boot ו-Thymeleaf.

---

## 3. הבעיה שהמערכת פותרת

### בעיות קיימות בשוק:
| בעיה | תיאור |
|------|--------|
| תהליך הזמנה מסורבל | לקוחות נאלצים להתקשר, לחכות ולמלא טפסים מנייר |
| חוסר שקיפות | מחירים לא ברורים, תוספות נסתרות |
| ניהול ידני | חברות משתמשות בגיליונות אקסל לניהול הצי |
| חוסר ניראות | מנהלים חסרים מידע real-time על מצב הצי |
| חווית משתמש ירודה | אתרים ישנים, לא ריספונסיביים |
| ניהול נזקים עלוב | תיעוד לקוי של נזקים גורם לסכסוכים |

### הפתרון של DriveFlow:
- הזמנה עצמאית מלאה 24/7 דרך האינטרנט
- מחשבון מחיר שקוף בזמן אמת
- ניהול דיגיטלי מלא של הצי, הזמנות ולקוחות
- לוח מחוונים למנהל עם מדדים עסקיים חיים
- תיעוד דיגיטלי של מצב הרכב בכל עסקה

## 4. מטרות המערכת

### מטרות עסקיות:
1. הגדלת כמות ההזמנות באמצעות ממשק עצמאי ונוח
2. צמצום עלויות תפעול ידני
3. שיפור ניהול הצי ואחוזי הניצולת

5. שיפור שביעות רצון לקוחות

### מטרות טכניות:
1. בניית מערכת מודולרית וניתנת להרחבה
2. ארכיטקטורה נקייה ומתועדת
3. ממשק ריספונסיבי לכל מכשיר
4. אבטחת מידע ברמה גבוהה
5. ביצועים גבוהים — עמידה בעומסי שיא

## 6. חזון המוצר

> "להפוך את חווית השכרת הרכב לפשוטה, שקופה ומהנה כמו הזמנת מלון — עם ניהול עסקי חכם שמאפשר לחברות להצמיח את העסק ולספק שירות יוצא דופן."

DriveFlow שואפת להיות המערכת המועדפת על חברות השכרת רכב בינוניות-גדולות, המאפשרת ניהול מלא של תהליך ההשכרה מקצה לקצה.

## 7. גבולות המערכת (Scope)

### כלול (In Scope):
- ✅ הרשמה, התחברות וניהול פרופיל לקוח
- ✅ חיפוש רכבים לפי תאריך, מיקום וקטגוריה
- ✅ הזמנת רכב עם בחירת תוספות
- ✅ מחשבון מחיר בזמן אמת
- ✅ עיבוד תשלום (Simulated)
- ✅ אישורי הזמנה והודעות במייל
- ✅ ניהול ציים רכבים (CRUD)
- ✅ ניהול לקוחות וקבלת דוחות
- ✅ ניהול תחזוקה ונזקים
- ✅ Dashboard מנהל עם KPIs
- ✅ ניהול סניפים
- ✅ דוחות עסקיים

### לא כלול (Out of Scope):
- ❌ אפליקציית מובייל native
- ❌ שילוב עם ספקי תשלום אמיתיים (רק Simulation)
- ❌ מערכת GPS/מעקב רכבים בזמן אמת
- ❌ מערכת ביטוח רכב
- ❌ API פתוח לצדדים שלישיים
- ❌ Multi-language (רק עברית)

## 8. MVP — גרסה ראשונה

הגרסה הראשונה של DriveFlow תכלול:

### Core Features (MVP):
1. **Authentication** — הרשמה, התחברות, ניהול פרופיל
2. **Vehicle Search** — חיפוש רכבים לפי תאריך ומיקום
3. **Booking Flow** — תהליך הזמנה מלא (בחירה → תוספות → תשלום → אישור)
4. **Customer Area** — אזור אישי עם היסטוריית הזמנות
5. **Admin Dashboard** — ניהול בסיסי של רכבים, הזמנות ולקוחות
6. **Vehicle Management** — CRUD לניהול ציי רכבים

## 10. אילוצים וסיכונים

### אילוצים טכניים:
- Java 17+, Spring Boot 3.x
- PostgreSQL / MySQL
- Thymeleaf (Server-side rendering)
- Bootstrap 5

### סיכונים:
| סיכון | רמה | מיטיגציה |
|-------|------|-----------|
| עיכוב בשלב התשלום | בינונית | שימוש ב-Simulated Payment בלבד |
| מורכבות קלנדר | גבוהה | שימוש בספריית date-picker מוכחת |
| נפח נתונים גדול | נמוכה | Pagination בכל רשימות הנתונים |
| חוסר ניסיון ב-Spring Security | בינונית | שימוש ב-Basic Form Login עם JWT |
