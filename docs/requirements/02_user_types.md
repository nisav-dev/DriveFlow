# DriveFlow — סוגי משתמשים והרשאות

---

## 1. לקוח (CUSTOMER)

### תיאור תפקיד:
משתמש קצה המשתמש בפלטפורמה לחיפוש, הזמנה וניהול השכרות רכב אישיות.
יכול להיות לקוח פרטי או נציג עסקי.

### פרופיל טיפוסי:
- גיל: 21+
- נוסע עסקי, תייר, אדם שרכבו בתיקון
- משתמש מחשב/מובייל

### הרשאות:
| פעולה | מותר |
|--------|-------|
| הרשמה והתחברות | ✅ |
| עריכת פרופיל אישי | ✅ |
| חיפוש רכבים | ✅ |
| צפייה בפרטי רכב | ✅ |
| יצירת הזמנה | ✅ |
| ביטול הזמנה (בתנאים) | ✅ |
| צפייה בהיסטוריית הזמנות | ✅ |
| הורדת קבלות | ✅ |
| שליחת בקשת שירות | ✅ |
| צפייה בנתוני לקוחות אחרים | ❌ |
| ניהול רכבים | ❌ |
| גישה ל-Admin Dashboard | ❌ |

### פעולות אפשריות:
1. **הרשמה** — יצירת חשבון עם אימות מייל
2. **חיפוש** — חיפוש רכבים לפי תאריך, מיקום, קטגוריה
3. **הזמנה** — בחירת רכב, תוספות, ומילוי פרטי הזמנה
4. **תשלום** — תשלום מקוון עם אישור
5. **ניהול הזמנות** — צפייה, ביטול, שינוי הזמנות
6. **פרופיל** — עדכון פרטים אישיים ורישיון נהיגה
7. **תמיכה** — שליחת פנייה לשירות לקוחות

---

## 2. מנהל מערכת (ADMIN)

### תיאור תפקיד:
משתמש בעל הרשאות מלאות לניהול המערכת כולה.
אחראי על תצורת המערכת, ניהול משתמשים, ניהול הצי ודוחות מנכ"ל.

### פרופיל טיפוסי:
- מנהל IT / מנהל תפעול ראשי
- גישה מלאה לכל חלקי המערכת

### הרשאות:
| פעולה | מותר |
|--------|-------|
| כל הרשאות לקוח | ✅ |
| ניהול משתמשים (CRUD) | ✅ |
| ניהול רכבים (CRUD) | ✅ |
| ניהול סניפים (CRUD) | ✅ |
| ניהול קטגוריות (CRUD) | ✅ |
| ניהול כל ההזמנות | ✅ |
| עדכון סטטוס הזמנה | ✅ |
| ניהול תחזוקה | ✅ |
| צפייה בדוחות כלכליים | ✅ |
| ייצוא נתונים | ✅ |
| ניהול הרשאות נציגים | ✅ |
| הגדרות מערכת | ✅ |
| מחיקת נתונים | ✅ |

### פעולות אפשריות:
1. **Dashboard ראשי** — KPIs, גרפים, התראות
2. **ניהול ציים** — הוספה, עריכה, מחיקה, שינוי סטטוס רכבים
3. **ניהול הזמנות** — צפייה, עדכון, ביטול, אישור כל ההזמנות
4. **ניהול לקוחות** — צפייה בפרופילים, עריכה, חסימה
5. **ניהול תחזוקה** — תזמון שירות, עדכון רשומות
6. **ניהול נזקים** — תיעוד נזקים, צפייה בדוחות
7. **דוחות** — הכנסות, ניצולת, לקוחות, הזמנות
8. **הגדרות** — ניהול סניפים, קטגוריות, תעריפים
9. **ניהול נציגים** — הוספת/עדכון חשבונות נציגי שירות

## 3. נציג שירות (AGENT)

### תיאור תפקיד:
עובד סניף הפועל בשם הלקוח לניהול הזמנות, תהליך איסוף והחזרת רכב.
בעל הרשאות מוגבלות ביחס למנהל, ממוקד בפעולות יומיומיות מול הלקוח.

### פרופיל טיפוסי:
- עובד קבלה בסניף
- אחראי על מסירת ומסירת רכבים
- מטפל בהזמנות ותלונות לקוחות

### הרשאות:
| פעולה | מותר |
|--------|-------|
| צפייה בהזמנות הסניף שלו | ✅ |
| עדכון סטטוס הזמנה (איסוף/החזרה) | ✅ |
| יצירת הזמנה בשם לקוח | ✅ |
| צפייה בפרטי לקוח | ✅ |
| עדכון מצב רכב | ✅ |
| דיווח נזק | ✅ |
| יצירת רשומת תחזוקה | ✅ |
| מחיקת הזמנות | ❌ |
| גישה לדוחות כלכליים | ❌ |
| ניהול רכבים בסניפים אחרים | ❌ |
| ניהול משתמשים | ❌ |
| הגדרות מערכת | ❌ |

### פעולות אפשריות:
1. **איסוף רכב** — אישור פרטי לקוח, תיעוד מצב רכב, מסירת מפתחות
2. **החזרת רכב** — בדיקת רכב, תיעוד נזקים, סגירת חשבון
3. **ניהול הזמנות** — יצירה, עדכון, ביטול הזמנות לסניף
4. **תמיכה ללקוח** — עדכון פרטי הזמנה, טיפול בבקשות
5. **דיווח תחזוקה** — רישום בעיות, תיאום שירות
6. **צפייה בלוח שנה** — מצב רכבים ב-Queue לאיסוף/החזרה
